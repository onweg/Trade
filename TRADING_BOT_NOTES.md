## Минимальная версия продукта (MVP) и логика текущего этапа

### Цель этапа
Построить минимально рабочий каркас серверного трейдинг-бота на C++ с модульной архитектурой, чтобы:
- Быстро запускать и тестировать новые торговые стратегии как плагины.
- Легко расширять и поддерживать проект.
- Чётко разделять бизнес-логику, инфраструктуру и интерфейсы.

### Минимальный результат (MVP)
- Есть модели (Order, Trade, MarketData) и интерфейсы (IStrategy, IExchangeApi).
- Есть заглушки бизнес-логики (TradeEngine, OrderManager).
- Есть заглушки инфраструктуры (ExchangeApi, Logger).
- Есть пример стратегии (DummyStrategy) и загрузчик плагинов (PluginLoader).
- Всё это собирается и запускается через main.cpp, где:
    - Инициализируются основные компоненты.
    - Загружается стратегия как плагин.
    - Имитация получения рыночных данных и передачи их в стратегию.

### Краткая логика взаимодействия компонентов
1. **main.cpp** — точка входа, инициализация всех компонентов.
2. **PluginLoader** — загружает стратегию (например, DummyStrategy) как динамическую библиотеку.
3. **TradeEngine** — принимает рыночные данные, передаёт их в стратегию.
4. **OrderManager** — управляет ордерами, которые создаёт стратегия.
5. **ExchangeApi** — имитирует отправку ордеров на биржу.
6. **Logger** — пишет логи о действиях системы.
7. **Strategy (плагин)** — реализует интерфейс IStrategy, реагирует на рыночные данные, создаёт ордера.

### Пример сценария работы MVP
- main.cpp загружает DummyStrategy через PluginLoader.
- main.cpp имитирует получение рыночных данных (MarketData) и передаёт их в TradeEngine.
- TradeEngine вызывает onMarketData у стратегии.
- Стратегия решает, нужно ли создать ордер, и сообщает OrderManager.
- OrderManager передаёт ордер в ExchangeApi.
- Logger пишет, что произошло.

## **Архитектура папок для C++ серверного трейдинг-бота (модульная + чистая архитектура, с разделением моделей и интерфейсов)**

```
TradeBot/
│
├── CMakeLists.txt           # Файл сборки проекта (основной CMake для всего проекта)
├── README.md                # Описание проекта, инструкция по запуску
├── PROJECT_GUIDE.md         # Гайд по проекту и организационные вопросы
│
├── include/                 # Публичные заголовочные файлы
│   └── tradebot/
│        ├── models/                # Модели (Order, Trade, MarketData)
│        │     ├── Order.h          # Order — модель ордера
│        │     ├── Trade.h          # Trade — модель сделки
│        │     └── MarketData.h     # MarketData — рыночные данные
│        ├── interfaces/            # Интерфейсы (IStrategy, IExchangeApi и др.)
│        │     ├── IStrategy.h      # IStrategy — интерфейс для стратегий
│        │     └── IExchangeApi.h   # IExchangeApi — интерфейс для работы с биржей
│        └── infrastructure/        # Публичные инфраструктурные компоненты
│              └── PluginLoader.h   # PluginLoader — интерфейс/реализация загрузчика плагинов
│
├── core/                    # Бизнес-логика (ядро)
│   ├── OrderManager.h       # OrderManager — управление ордерами (создание, отмена, хранение)
│   ├── TradeEngine.h        # TradeEngine — обработка рыночных данных и принятие торговых решений
│   └── ...
│
├── infrastructure/          # Инфраструктурные компоненты (реализация)
│   ├── ExchangeApi.h        # ExchangeApi — взаимодействие с биржей, реализация IExchangeApi
│   ├── Logger.h             # Logger — базовые методы логирования (info, warn, error)
│   └── ...
│
├── plugins/                 # Плагины-стратегии (динамические библиотеки, приватные или публичные)
│   └── DummyStrategy/           # Пример стратегии (заглушка)
│        ├── DummyStrategy.cpp   # DummyStrategy — пример реализации стратегии
│        └── CMakeLists.txt      # CMake для сборки плагина-стратегии
│   └── ...
│
├── src/                     # Точка входа и glue-код
│   ├── main.cpp                 # main.cpp — запуск приложения, инициализация компонентов
│   ├── PluginLoader.cpp         # PluginLoader.cpp — реализация загрузчика плагинов
│   └── ...
│
├── tests/                   # Тесты
│   └── ...                  # Модульные и интеграционные тесты
│
├── scripts/                 # Скрипты для сборки, деплоя, анализа
│   └── ...                  # Вспомогательные скрипты
│
├── config/                  # Примеры конфигов (без секретов!)
│   └── config.example.json  # Пример конфигурационного файла
│   └── ...
│
├── docs/                    # Документация
│   └── ...                  # Описание архитектуры, API, туториалы
│
└── third_party/             # Сторонние библиотеки (если не через пакетный менеджер)
    └── ...
```

**Пояснения:**
- `include/tradebot/models/` — все основные сущности (Order, Trade, MarketData).
- `include/tradebot/interfaces/` — все интерфейсы (IStrategy, IExchangeApi и др.).
- `include/tradebot/infrastructure/PluginLoader.h` — инфраструктурный компонент для загрузки плагинов. Он не является ни моделью, ни бизнес-интерфейсом, поэтому вынесен в отдельную папку для лучшей читаемости и структурированности кода.
- Остальные папки — как и ранее, для бизнес-логики, инфраструктуры, плагинов, тестов и т.д.
